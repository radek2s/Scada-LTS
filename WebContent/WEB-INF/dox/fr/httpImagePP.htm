Le URL est l'adresse de l'image à récupérer. Utilisez l'icône pour tester l'URL. Les champs Temps d'arrêt et réessayer déterminent combien de temps il faut attendre que le serveur web renvoie l'image, et combien de fois il faut réessayer la connexion en cas d'échec. Si toutes les tentatives sont épuisées, un événement d'exception de point de données est déclenché.

Les images peuvent être automatiquement mises à l'échelle selon les besoins pour économiser de la mémoire ou de l'espace dans le système de fichiers. (Les images peuvent également être mises à l'échelle, mais comme cela n'augmente pas la résolution, ce n'est généralement pas pratique). Le type de mise à l'échelle en "pourcentage" est une méthode de mise à l'échelle "relative" qui donne une image qui est mise à l'échelle à la taille Pourcentage d'échelle de l'image originale. La méthode d'échelle "box" est une méthode d'échelle "absolue" qui met à l'échelle l'image originale de manière à ce qu'elle s'inscrive dans la "box" définie par les valeurs Largeur d'échelle et Hauteur d'échelle données.

La limite de lecture permet de fixer une taille maximale de réponse pouvant être reçue. Si la taille des données de l'image dépasse la valeur donnée, les données seront rejetées et un événement de récupération des données sera déclenché.

Si la source de l'image est une webcam, vous pouvez entrer le code du flux en direct dans la zone de texte prévue à cet effet. Ce code sera saisi mot pour mot dans une nouvelle fenêtre ou un nouvel onglet du navigateur (entouré des balises html et body). Si le flux en direct de la webcam nécessite l'utilisation d'un composant ActiveX ou d'un applet Java, vous devrez installer ces composants de manière appropriée dans l'instance Mango.
