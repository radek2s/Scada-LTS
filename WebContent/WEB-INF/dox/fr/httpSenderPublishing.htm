Vue d'ensemble

L'éditeur expéditeur HTTP envoie chaque mise à jour de point (ou changement, selon le cas) à une URL de destination donnée.

HTTP propriétés de l'expéditeur

La sélection Méthode détermine le type de méthode HTTP qui sera utilisé pour fournir les données. Les données envoyées sont identiques dans les deux cas ; la différence est une question de structure du message. La sélection URL définit la ressource à laquelle envoyer les données. Les schémas valides sont "http" et "https". Le serveur peut être un nom de domaine ou une adresse IP. Les chemins d'accès et les chaînes de requête sont également valides.

Une liste de En-têtes statiques et Paramètres statiques peut également être ajoutée. Ces données sont incluses avec chaque requête envoyée à l'URL. Il s'agit donc d'un endroit approprié pour inclure des références d'autorisation ou d'autres artefacts d'identification. Pour ajouter un en-tête et un paramètre statique, entrez sa clé et sa valeur, puis cliquez sur l'icône . Pour supprimer un en-tête ou un paramètre statique existant, cliquez sur l'icône associée à l'entrée.

La sélection Date format détermine comment l'éditeur formatera les horodatages associés à des valeurs ponctuelles. Le format "de base" est de la forme "yyyyMMddHHmmss". "Fuseau horaire" est similaire, mais comprend plus d'informations concernant la localisation de l'horodatage. Il est de la forme "yyyy-MM-dd'T'HH:mm:ss 'Z'". "UTC" utilise une valeur numérique qui représente le nombre de millisecondes écoulées depuis minuit le 1er janvier 1970.

Réponse du HTTP

Tout autre code de réponse que "200 OK" est considéré comme une erreur et une alarme sera déclenchée. En outre, tout contenu de réponse inclus avec une réponse OK est considéré comme une information d'avertissement. Le réglage Raise du résultat d'alerte contrôle si l'existence d'un contenu de réponse provoque le déclenchement d'une alarme d'information incluant le contenu du message d'événement.

Tests de configuration

Une demande de test peut être envoyée - en utilisant la configuration donnée mais en incluant uniquement des paramètres statiques - en cliquant sur le bouton "Envoyer les paramètres statiques". Le résultat de la tentative d'envoi sera affiché sous les boutons. Pour annuler un test avant qu'il ne s'arrête de lui-même, cliquez sur le bouton "Annuler".

Points publiés

La section Points énumère les points à publier. La publication des points peut être activée () ou désactivée (). Désactiver un point permet de ne pas l'utiliser sans avoir à le supprimer explicitement. Le nom du paramètre est la clé qui sera utilisée pour le point dans les paramètres de la requête. La valeur sera la valeur actuelle du point. Sélectionnez Inclure l'horodatage pour inclure dans la demande un paramètre qui contient l'heure de mise à jour de la valeur. La clé pour l'heure sera "__time", et la date sera encodée selon la valeur du format de date. (Voir la documentation "Formats de date/heure" pour plus d'informations).
