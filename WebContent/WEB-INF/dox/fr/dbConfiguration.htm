Le Scada-LTS permet actuellement l'utilisation de deux systèmes de gestion de base de données (SGBD): Derby and MySQL.

Le Derby est utilisé par défaut par le système. Toutefois, si vous avez besoin de modifier sa configuration initiale, vous pouvez le faire en cliquant sur le bouton est nécessaire pour modifier le fichier situé dans "Scada-LTS*/WEB-INF/classes/env.properties" et "Scada-LTS*/WEB-INF/classes/env.properties.derby".

Pour utiliser MySQL, vous devez saisir les données nécessaires à son utilisation correcte. Ainsi, le fichier devait être modifié est en "Scada-LTS*/WEB-INF/classes/env.properties.mysql". Fondamentalement, les paramètres suivants doivent être saisis :

db.Nomd'utilisateur: Nom d'utilisateur.

db.Motdepasse: Mot de passe.

Vous avez fait les modifications nécessaires, vous devez choisir MySQL comme option et cliquer sur l'icône pour enregistrer. Ensuite, redémarrez le serveur d'application (par exemple Tomcat).

Scada-LTS*: Scada-LTS l'emplacement sur le serveur d'application (ex : "Pasta_Tomcat/webapps/Scada-LTS").
