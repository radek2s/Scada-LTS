Vue d'ensemble

La source de métadonnées est ainsi nommée parce qu'elle est capable de combiner des points existants en de nouveaux points. Plutôt que de tirer ses informations d'une source externe, elle utilise les valeurs d'autres points, ce qui permet de les manipuler de manière arbitraire.

JavaScript / ECMAScript

Cette source de données nécessite une certaine connaissance de JavaScript, également connu sous le nom d'ECMAScript. (La spécification complète de l'ECMAScript est disponible ici). Le JavaScript est sans doute le langage de script le plus populaire utilisé aujourd'hui, c'est le seul langage disponible pour tous les navigateurs web populaires. Il en résulte que, de loin, l'utilisation la plus courante de JavaScript est celle des pages web. Le contexte dans la source de métadonnées est bien sûr différent, il n'y a pas d'objets "fenêtre" ou "document", entre autres différences plus subtiles. Mis à part les différences contextuelles, une implémentation complète de l'ECMAScript est disponible dans cette source de données. Plus précisément, l'implémentation Rhino est utilisée.
