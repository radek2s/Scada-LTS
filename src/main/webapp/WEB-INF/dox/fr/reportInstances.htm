Vue d'ensemble

Les rapports en Scada-LTS ont deux représentations : modèles qui définissent le contenu d'un rapport, et instances qui sont le résultat de l'exécution d'un rapport avec un modèle donné. Simplement : le moteur de rapport crée des instances à partir de modèles. Un modèle possède des attributs comme un nom, une liste de points qu'il inclut et la plage de dates des valeurs qu'il extrait. Une instance héritera de son nom du modèle, mais aura ses propres attributs tels que l'heure à laquelle elle a été exécutée, la durée d'exécution, le nombre d'enregistrements qu'elle inclut et, bien sûr, les données.

Une instance peut être considérée comme un instantané des données au moment où elle a été exécutée. Les attributs de l'instance ne peuvent pas être modifiés. De plus, les données que l'instance rapporte existent tant que l'instance existe. (C'est-à-dire qu'elles survivent à la purge des données sources.) C'est pourquoi il est important de purger les instances de rapport - en particulier les grandes instances - lorsqu'elles ne sont plus nécessaires car elles peuvent consommer de grandes quantités de stockage.

Les modèles de rapport - et les instances qu'ils créent - appartiennent à l'utilisateur qui les a créés. Ils ne peuvent pas être explicitement partagés dans le système, mais peuvent l'être implicitement. Voir la documentation "Modèles de rapport" pour plus d'informations.

Piste des rapports

La file d'attente des rapports énumère toutes les instances de rapport pour l'utilisateur actuel. Cette liste ne se rafraîchit pas automatiquement. Un rafraîchissement manuel de la liste peut être déclenché en cliquant sur l'icône . Certains événements, cependant, déclenchent également un rafraîchissement de la liste, comme l'exécution d'un rapport.

Les instances de rapport dans n'importe quel état seront listées, qu'elles soient terminées ou en cours d'exécution. Les colonnes de la liste refléteront l'état actuel de l'instance.

Les colonnes de la liste ont les définitions suivantes:

  • Nom du rapport : le nom hérité du modèle de rapport. Une fois l'instance créée, le nom du rapport ne change pas, même si le nom du modèle est modifié.
  • Début de la durée d'exécution : l'heure à laquelle le moteur de rapport a commencé à créer l'instance de rapport
  • Run duration : le temps nécessaire à la création de l'instance
  • de : l'heure à partir de laquelle les enregistrements du rapport ont été sélectionnés (inclus)
  • Vers : l'heure jusqu'à laquelle les enregistrements du rapport ont été sélectionnés (exclusif)
  • Records : le nombre total d'enregistrements sélectionnés

La case à cocher Ne pas purger permet aux utilisateurs d'empêcher la suppression d'instances de rapport importantes par le processus de purge programmé. (Voir la documentation "Autres paramètres" pour plus d'informations sur la purge des rapports). Cette fonction ne doit être utilisée qu'en cas de nécessité, car les instances de rapport peuvent potentiellement consommer de grandes quantités de stockage.

La dernière colonne du tableau fournit des contrôles pour la gestion des instances. L'icône lance le téléchargement d'un fichier CSV des données du rapport, qui peut être chargé dans un tableur. L'icône ouvre une nouvelle fenêtre de navigateur affichant des informations sur les instances du rapport, des statistiques sur les données et un graphique des données du rapport. Un clic sur l'icône supprime l'instance du rapport. Notez que la suppression des grandes instances de rapport peut prendre un certain temps.
