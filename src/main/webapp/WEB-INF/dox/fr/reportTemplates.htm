Vue d'ensemble

Modèles de rapport fournit la définition de la manière de créer des instances de rapport. Pour une description des modèles par rapport aux instances, voir la documentation "Instances de rapport".

Liste des modèles de rapport

Pour ajouter un nouveau modèle de rapport, cliquez sur l'icône . Pour modifier un modèle existant, cliquez sur le nom du modèle dans la liste "Modèles de rapport".

Edition des détails du modèle de rapport

Le modèle nom est utilisé pour référencer visuellement le modèle. Il est recommandé d'utiliser un nom unique pour chaque détecteur, mais ce n'est pas obligatoire.

Utilisez la liste Points pour sélectionner les points à inclure dans le rapport. Pour ajouter un point, sélectionnez-le dans la liste et cliquez sur l'icône . Pour supprimer un point existant, cliquez sur l'icône icône associée au point.

Les valeurs Couleur peuvent être un hexagone de style HTML, ou tout nom de couleur dans la spécification HTML. Par exemple, "#2468AC", "red", et "dodgerblue" sont tous des valeurs valides.

La plage de dates est utilisée pour déterminer les valeurs à sélectionner pour le rapport. L'intervalle peut être relatif ou absolu. Dans la plupart des cas, une fourchette relative est appropriée. Les plages de dates relatives peuvent être basées soit sur des données "antérieures", soit sur des données "passées". Une plage passé comprend la période donnée se terminant maintenant. Une plage précédente comprend également la période donnée, mais son heure de fin est quantifiée pour correspondre au type de période. Par exemple, si le type de période est "Heure(s)" et que le nombre de périodes est 1, et que le rapport se déroule à 18:05, la période qui sera utilisée est de 17:00 (inclus) à 18:00 (exclus). Si le nombre de périodes était, par exemple, de 3, la période serait de 15h00 à 18h00. De même, le "Mois" commence à minuit le premier jour du mois précédent et se termine le dernier jour du mois précédent (lorsque le nombre de périodes est 1). Les autres types de périodes fonctionnent de la même manière. Une semaine commence le lundi à minuit, conformément aux normes ISO.

Programmation des rapports

Les rapports peuvent être programmésd pour s'exécuter automatiquement. Utilisez la sélection Exécuter chaque... pour déterminer un simple événement temporel sur lequel il doit s'exécuter, ou définissez un modèle de cron pour un contrôle plus spécifique. (Voir la documentation "Motifs de cron" pour plus d'informations.) Les événements temporels se produisent au début de la période donnée. Les semaines commencent le lundi conformément aux normes ISO. Un délai d'exécution peut être appliqué si les données attendues sur le rapport ont tendance à être collectées plus tard que le temps d'exécution absolu.

Les rapports prévus pour les utilisateurs handicapés ne sont pas exécutés.

Rapport sur l'envoi de courriers électroniques

Bien que les rapports ne puissent pas être explicitement partagés avec le système, ils peuvent l'être implicitement en créant une liste de diffusion à laquelle envoyer l'instance de rapport générée. Le contenu du courrier électronique est le même que celui de la fenêtre "Report Chart" ouverte dans le panneau "Report queue". Pour inclure le fichier d'exportation CSV avec le courriel, cochez la case Inclure les données tabulaires.

Sélectionnez les Destinataires du courrier électronique auxquels envoyer le courrier électronique du rapport. Les destinataires peuvent être des listes de diffusion, des utilisateurs du système ou des adresses électroniques libres. Cliquez sur l'icône pour envoyer un message de test aux destinataires sélectionnés.

Important : Les rapports envoyés par courrier électronique sont automatiquement supprimés immédiatement après leur envoi.

Gestion du modèle

Pour enregistrer un modèle de rapport, cliquez sur l'icône en haut à droite du panneau des critères de rapport. Pour supprimer un modèle existant, cliquez sur l'icône . Pour exécuter un modèle de rapport immédiatement, cliquez sur l'icône . Notez que certains modèles de rapport peuvent contenir beaucoup d'informations et peuvent donc prendre beaucoup de temps à exécuter. Ainsi, tous les rapports sont exécutés de manière asynchrone par rapport à l'interface utilisateur. Pour faire une copie d'un modèle de rapport, cliquez sur l'icône .
