Les attributs utilisés pour configurer les points de source de données SQL diffèrent légèrement selon que la requête est basée sur des lignes ou des colonnes. Dans les deux cas, le type de données Data type détermine le type de données du point de données Scada-LTS. Les valeurs de l'ensemble des résultats sont automatiquement converties dans ce type.

Pour les requêtes basées sur les lignes, l'identificateur de ligne Row identifier est comparé à la première colonne du jeu de résultats pour trouver la ou les lignes qui contiennent des valeurs (la deuxième colonne) pour ce point. Les dépassements de temps sont pris dans la troisième colonne si elle existe.

Pour les requêtes basées sur des colonnes, le Nom de la colonne correspond au nom de la colonne dans l'ensemble de résultats dont la valeur du point est extraite. La colonne Colonne d'annulation du temps (facultatif) spécifie la colonne à utiliser comme annulation du temps du point. Notez que plusieurs points peuvent utiliser la même dérogation temporelle le cas échéant.

Mise à jour des déclarations

Une Déclaration de mise à jour peut éventuellement être définie pour un point, ce qui détermine comment la valeur du point peut être fixée dans la base de données. Il s'agit d'une instruction SQL standard utilisant "update" ou "insert". Si cette instruction n'est pas définie, le point sera considéré comme non paramétrable.
